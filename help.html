Overview

NodeBook is a powerful, browser-based knowledge graph editor that allows you to visualize, create, and manipulate complex relationships between nodes (entities) and edges (relationships).

Key Features

üé® Interactive Visualization

Force-directed graph layout with customizable node colors, sizes, and edge relationships.

üìù Rich Editing

Add, edit, and delete nodes and edges with full property management and inline editing capabilities.

üîç Search & Filter

Powerful search functionality and advanced filtering options to focus on specific parts of your graph.

‚ö° Shortest Path

Find the shortest path between any two nodes or discover all possible paths.

‚Ü∂ Undo/Redo

Full history management with up to 50 levels of undo/redo.

üîí Privacy First

All processing happens in your browser. No data is sent to servers.

Getting Started

Creating Your First Graph

  1. Click the File tab in the left sidebar
  2. Click New to start with a blank canvas
  3. Switch to the Tools tab
  4. Start adding nodes and connections using the context menu (right-click on canvas)

Opening an Existing Graph

  1. Click File ‚Üí Open
  2. Select a JSON file from your computer
  3. Alternatively, drag and drop a JSON file directly onto the canvas
üí° Tip: Sample graphs are available in the JSON Samples folder for testing and learning.

Tools & Features

Tools Tab Functions

‚úã Pan (Hold Space)

Activates pan mode. Click and drag to move around the canvas. You can also hold the spacebar while panning without activating the tool.

‚ùÑÔ∏è Freeze (F)

Toggles the physics simulation:

  • Frozen: Nodes stay in place, preventing automatic rearrangement
  • Active: Force-directed layout continuously adjusts node positions

‚äû Shuffle

Restarts the force simulation with a fresh layout. Useful when nodes overlap or the layout becomes cluttered.

‚ö° Shortest Path

Finds paths between nodes:

  1. Click the Path button
  2. Click on a starting node
  3. Click on a destination node
  4. Choose:
    • Shortest Path: Displays the fastest route
    • All Paths: Shows all possible routes between nodes

‚ûï Add Graph

Merges another graph file into your current graph:

  • Opens a file picker to select a JSON graph file
  • Adds all nodes and edges from the selected file to your current graph
  • Useful for combining multiple data sources or building composite graphs
  • Maintains all properties and relationships from both graphs

‚öôÔ∏è Force Strength Control

Adjusts the strength of the force-directed layout simulation:

  • Slider Range: 50 (Weak) to 500 (Strong)
  • Default: 200 - balanced for most graphs
  • Weak Force: Nodes stay closer together, more compact layout
  • Strong Force: Nodes spread out more, better separation for dense graphs
  • Adjust in real-time to find the optimal layout for your graph

Editing Nodes & Edges

Context Menu (Right-Click)

Right-clicking on different areas opens context-specific menus:

Canvas Context Menu

  • Add Node Here: Creates a new node at the cursor position
  • Paste Node: Pastes a previously copied node

Node Context Menu

  • Edit Properties: Opens the properties panel
  • Pin/Unpin: Fixes the node position or allows it to move with simulation
  • Copy Node: Copies the node for pasting elsewhere
  • Connect to Another Node: Starts connection mode
    1. Right-click a node and select "Connect to Another Node"
    2. Click on the target node to connect
    3. Enter relationship details in the dialog
  • Delete Node: Removes the node and all its connections

Edge Context Menu

  • Edit Edge: Opens edge properties
  • Reverse Direction: Swaps source and target nodes
  • Toggle Direction: Changes between directed (‚Üí) and undirected (‚Äî)
  • Delete Edge: Removes the connection

Properties Panel

Click on any node or edge to open the properties panel on the right side.

Node Properties

  • Name: The primary identifier
  • Description: Detailed information about the node
  • Color: Visual appearance (click swatch to change)
  • Size: Node radius (5-50 pixels)
  • Type: Category or classification
  • Custom Properties: Add unlimited key-value pairs

Edge Properties

  • Name: Edge identifier
  • Relationship: The type of connection
  • Source: Starting node
  • Target: Ending node
  • Directed: Whether the edge has direction (arrow)
  • Color: Edge color
  • Weight: Relationship strength or importance

Privacy & Data Security

üîí Your Data Stays Private

NodeBook is designed with privacy as a core principle:

100% Client-Side

All processing happens in your browser. No server communication required for core functionality.

No Data Collection

We don't collect, store, or transmit any of your graph data, files, or personal information.

Local Storage Only

Your preferences and settings are stored locally in your browser using localStorage.

Open Source

Our code is transparent and can be audited by anyone.

Privacy Banner

On first visit, you'll see a privacy banner at the top of the page explaining that NodeBook runs entirely in your browser. You can:

  • Click "Got it" to dismiss the banner - your choice is remembered
  • Click the privacy icon (üîí Private & Local) in the status bar to view privacy details anytime

What Gets Stored Locally?

Only these preferences are saved in your browser's localStorage:

  • Privacy banner dismissal status
  • No graph data or file content is ever stored automatically
üí° Important: You are responsible for saving your work. Use File ‚Üí Save to download your graphs as JSON files on your device.

Tips & Best Practices

Performance

  • Freeze for Large Graphs: Graphs with 100+ nodes benefit from freezing the simulation
  • Use Filter: Hide unnecessary nodes to improve performance
  • Adjust Force Strength: Lower force strength for better performance with large graphs

Organization

  • Use Colors: Assign consistent colors to node types for easy identification
  • Meaningful Names: Use descriptive names for nodes and relationships
  • Add Descriptions: Include detailed descriptions for complex nodes
  • Pin Important Nodes: Pin key nodes to keep them in specific positions

Workflow

  • Save Frequently: Use Ctrl+S often to avoid losing work
  • Use Versions: Save As to create different versions of your graph
  • Export Backups: Export JSON regularly as backup
  • Test with Samples: Practice with sample graphs before creating complex graphs

Advanced Techniques

  • Explode for Analysis: Use the Explode feature to break down complex structures
  • Excel Integration: Use Excel converter for bulk data import
  • Path Analysis: Use "All Paths" to discover hidden connections
  • Custom Properties: Add metadata for advanced filtering and analysis

Troubleshooting

Common Issues

Graph Won't Load

  • Verify JSON format is correct
  • Check for syntax errors in the JSON file
  • Ensure file size is reasonable (under 10MB recommended)
  • Try opening the file in a text editor to verify it's valid JSON

Performance Issues

  • Freeze the simulation (press F)
  • Use the Filter function to hide unnecessary nodes
  • Reduce force strength
  • Close other browser tabs
  • Try a different browser (Chrome recommended)

Can't Find a Node

  • Use Search (Ctrl+F) to locate nodes by name
  • Check if filters are active
  • Try shuffling the layout
  • Zoom out to see the entire graph

Changes Not Saving

  • Click Save (Ctrl+S) after making changes
  • Check browser permissions for file downloads
  • Verify you have write permissions to the save location

Accidental Deletion

  • Immediately press Ctrl+Z to undo
  • Check if you have a recent save to restore from
